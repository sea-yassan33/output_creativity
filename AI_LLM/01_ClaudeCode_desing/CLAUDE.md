# プロジェクト概要

このプロジェクトはユーザの指示のもとHTML・CSS・JSファイルを生成します。
またその生成した設計書はドキュメント（Markdown形式）として出力します。

---

## 出力ルール（最重要・必ず守ること）

1. **ファイルを削除しない**
   - 作成済みのファイルは、理由の如何を問わず削除してはいけない。
   - 「不要になった」「古くなった」と判断した場合でも `rm` 等での削除は禁止。

2. **不要ファイルの扱い**
   - 不要と判断したファイルは削除せず、`_bk{YYMMDD}/` ディレクトリ配下に移動すること。
     - 例: 2026年6月20日に退避する場合 → `_bk260620/`
   - `_bk{YYMMDD}/` が存在しない場合は新規作成してから移動すること。
   - 同日内に複数回退避が発生した場合は、同じ `_bk{YYMMDD}/` ディレクトリにまとめて移動すること。

3. **出力方法**
   - このプロジェクトはHTML・CSS・JSファイルを生成します。生成物はsrc配下の`dev_{YYMMDDhhdd}/`ディレクトリを作成し、その配下に配置します
   - 設計書はdocs配下の`doc_{YYMMDDhhdd}/`に Markdown(.md) に出力します。
   - コードを生成するした際に、Markdown形式で設計・仕様をまとめてから出力すること。
   - 設計書にソースを記載する際はそのソースを`src/` ディレクトリ配下に対象ファイルのリンク先を記載すること。

4. **ソースコードの配置**
   - プログラムのソースコードが発生する場合は、必ず `src/` ディレクトリ配下に配置すること。
   - `src/` が存在しない場合は新規作成すること。
   - ドキュメント（.md）は `src/` には置かず、 `docs/` に置くこと。

---

## ディレクトリ構成

\```
first_projct/
├── CLAUDE.md
├── docs/ # 設計書（.md）の配置場所
├── src/  # ソースコードの配置場所
└── _bk{YYMMDD}/ # 不要ファイルの退避先（自動生成）
\```

## 作業時の振る舞い

- 新しいファイルを作る前に、既存ファイルと重複していないか確認すること。
- ファイルを「不要」と判断した場合、削除ではなく退避（上記ルール2）を行い、
  どのファイルをどこに移動したかを作業報告で明示すること。
- 出力するMarkdownは見出し構成を整え、誰が読んでも分かるようにすること。

## アクセス禁止ディレクトリ

- `.env`は環境変数の為、参照・読み込み・編集は一切禁止。
- `zz_dev/` 配下は開発中・検証用のディレクトリであり、参照・読み込み・編集は一切禁止。
